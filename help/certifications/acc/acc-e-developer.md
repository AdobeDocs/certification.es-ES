# Recorrido de certificación para Adobe [!DNL Campaign Classic] Experto en desarrollo

Obtenga información sobre cómo convertirse en un experto certificado en Adobe Developer.

## Experiencia mínima

Debe tener:

* Al menos dos años de experiencia en software de Campaign
* Al menos tres años de experiencia combinada en programación de JavaScript, desarrollo de SQL, incluida ETL, y tecnologías web (XML, HTML y servicios web)
* Experiencia en la ejecución de una campaña de marketing mediante un canal digital omnicanal
* Experiencia con arquitectura de bases de datos y conocimientos de marketing de bases de datos
* Capacidad de asignar requisitos a la funcionalidad de Campaign
* Capacidad para solucionar problemas en implementaciones de Campaign
* Comprensión básica de las configuraciones de servidor relacionadas con Campaign

>[!NOTE]
>
>Su certificación es válida durante dos años a partir del momento de su aprobación. Se recomienda utilizar la versión más reciente cuando sea posible, ya que cubre las capacidades de los productos más recientes.

```
Level: Expert (1-3 years' experience)
Passing Score: 34/50
Time: 100 mins
Delivery: Online proctored (requires camera access) or test center proctored
Available languages: English
Cost: $225 (global) / $150 (India)
Exam ID: AD0-E330
```

Complete el [cuestionario de preparación en línea](https://scorpion.caveon.com/launchpad/ad-q-e129-readiness-questionnaire-for-adobe-aem-assets-developer-professional-exam-copy-21ocv4/ad-q-e312-readiness-questionnaire-adobe-campaign-classic-developer-expert-exam){target="_blank"} para ver si su experiencia actual coincide con la cualificación mínima recomendada de los candidatos.

**Objetivos y ámbito del examen**

Sección 1: Configuración del sistema (16%)

* Aplicación de los ajustes de configuración y valores predeterminados de todo el sistema
* Dados varios parámetros, seleccione la función de Adobe Campaign adecuada para implementar un caso de uso
* Resolución de problemas de la configuración del sistema
* Implementación de configuraciones personalizadas
* Identificar las funciones de configuración de Campaign Panel de control de Campaign

Sección 2: Personalización (22%)

* Dados un conjunto de requisitos, aplique las prácticas recomendadas para diseñar un modelo de datos
* Describir la implementación de API en Campaign
* Describir las prácticas recomendadas para desarrollar con JavaScript en Adobe Campaign
* Aplicación de bibliotecas JavaScript en flujos de trabajo, llamadas SOAP, reglas de tipología y bloques de personalización
* Describir las prácticas recomendadas para utilizar funciones SQL en Adobe [!DNL Campaign Classic]
* En un escenario, comprenda cómo usar secuencias de comandos SQL en Adobe [!DNL Campaign Classic]
* Describir las funciones de OOTB más utilizadas

Sección 3: Gestión de acceso (8%)

* En un escenario, configure los parámetros de seguridad para un operador
* Describir cómo configurar las opciones de administración de acceso en función del documento de diseño de la solución
* Describir la administración de acceso mediante el panel de control de Campaign
* Identificar las limitaciones de la partición de datos y cuándo usar sysfilter

Sección 4: Integración de datos (16%)

* En un escenario, seleccione el mejor enfoque para integrar con fuentes de datos externas
* Prácticas recomendadas para el procesamiento de datos de flujo de trabajo
* Aplicación de técnicas de integración de datos en un flujo de trabajo
* Identificar la seguridad y la configuración de red que se integrarán con Adobe Campaign desde un sistema externo
* Configuración del proceso CRUD en un esquema desde fuera de Adobe Campaign

Sección 5: Mantenimiento de instancias (12%)

* Comprender y actuar en el mapa de calor del flujo de trabajo
* Demostrar el proceso para crear un flujo de trabajo personalizado o un informe, por ejemplo, monitorización
* Identificar problemas según los indicadores del sistema
* Explicar el impacto de los servicios y procesos en el sistema
* Demostrar métricas y comprobaciones básicas de auditoría del sistema

Sección 6: Configuración de campaña (25%)

* Aplicar técnicas de personalización avanzadas
* Configurar y aplicar asignaciones de destino personalizadas
* Identifique el uso apropiado de JavaScript en el flujo de trabajo de Campaign y por qué
* Aplicar código personalizado para manipular tablas temporales y contexto de flujo de trabajo
* Validación y depuración de la lógica del flujo de trabajo
* Aplicar configuración de envío avanzada
* Describir las prácticas de flujo de trabajo que tienen un impacto negativo en el rendimiento
* Dado un caso de uso, recomendar un diseño de flujo de trabajo de objetivos complejo

**Realizar una prueba de práctica**

Se recomienda realizar la prueba de práctica gratuita antes de comenzar. Tendrá acceso sin restricciones a todas las pruebas de práctica disponibles para los exámenes de nivel de experto de Adobe.

Nuestras pruebas de práctica se desarrollan desde el mismo modelo que los exámenes en directo. La prueba de práctica le ayudará a comprender los temas tratados en el examen en vivo.

Recuerde que aprobar una prueba de práctica no garantiza que se aprobará el examen de certificación.

Vaya a la [Portal de preparación de examen de certificación de Adobe](https://www.certmetrics.com/adobe/candidate/gmetrix_sso.aspx){target="_blank"} ahora, para realizar una prueba de práctica.

**Haga el examen de certificación**

Adobe utiliza dos proveedores para administrar los exámenes procesados, PSI y Ejemplos.

* **PSI** ofrece opciones de centro de pruebas y exámenes japoneses (si está disponible)
* **[!DNL Examity]** ofrece una opción en línea remota

Para programar, volver a programar o cancelar el examen, simplemente haga clic en el proveedor que desee y siga las instrucciones que aparecen en la pantalla. ¡Es así de simple!
