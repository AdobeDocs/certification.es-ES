# Recorrido de certificación para Adobe [!DNL Commerce] Experto para desarrolladores de Front-End

Obtenga información sobre cómo convertirse en un experto certificado en Adobe Developer.

## Experiencia mínima

Debe tener al menos 1-3 años de experiencia práctica con Adobe [!DNL Commerce] El desarrollo de vanguardia y la capacidad de traducir las necesidades expresadas por los requisitos de las tareas en una solución técnica.

Además, debe sentirse cómodo utilizando las siguientes tecnologías:

* Adobe [!DNL Commerce] - versión 2.4.3
* Comprensión básica de la estructura y el diseño del front-end
* Experiencia y comprensión de HTML, CSS (y preprocesadores), LESS, JavaScript y componentes de interfaz de usuario, jQuery, Underscore, Require.JS, Knockout, también plantillas PHP, conocimiento de base de datos y administración de contenido
* Familiaridad con la plataforma, Bloques de CMS, Widgets, Páginas de CMS, Páginas de categoría, Páginas de producto, Carro de compras y cierre de compra y Área del tablero de cuentas
* Comprensión básica o conocimiento de las configuraciones de administración
* Conocimiento sobre la ampliación y sobrescritura de estilos, diseños y archivos JS
* Ampliación de temas principales/jerarquía de temas
* Comando CLI (limpieza de caché, configuración:actualización, indexadores, modo de implementación) y Grunt
* Tecnologías de almacenamiento en caché, explorador del lado del cliente, comprensión de los almacenamientos (Cookies, almacenamiento local)
* Comprensión del diseño interactivo y las consultas de medios
* WYSIWYG/Generador de páginas
* Explicación de los paquetes de código
* Implementar la traducción
* Descripción del trabajo con plantillas de correo electrónico

>[!NOTE]
>
>Su certificación es válida durante dos años a partir del momento de su aprobación. Se recomienda utilizar la versión más reciente cuando sea posible, ya que cubre las capacidades de los productos más recientes.

**Objetivos y ámbito del examen**

Sección 1: Administración de temas (jerarquía de temas, configuración de imágenes, traducciones) (16%)

* Describir Adobe [!DNL Commerce] estructura de carpetas de temas y cómo se relaciona con temas basados en carpetas
* Demostrar la capacidad de crear un nuevo tema (Herencia/Abandonos, excepciones de diseño, propiedades de temas)
* Demostrar la capacidad de ampliar los temas existentes
* Demostrar la capacidad de personalizar correos electrónicos transaccionales
* Demostrar la capacidad de aplicar traducciones
* Dado un conjunto de casos de uso, identifique cuándo colocar archivos en la aplicación/código o la aplicación/diseño

Sección 2: Diseño XML y plantillas (plantillas phtml) (24%)

* Demostrar la capacidad de utilizar instrucciones XML de diseño
* Demostrar la capacidad de crear nuevos diseños de página
* Comprender la diferencia entre ampliar/combinar y anular XML
* Demostrar cómo pasar y utilizar argumentos en plantillas
* Demostrar la capacidad de crear y personalizar plantillas
* Aplicación de la seguridad de la plantilla (salida de escape)

Sección 3: Estilos (18%)

* Identifique el propósito de los estilos: m.less, style-l.less, print.less
* Describa la diferencia entre archivos parciales específicos (_extensions.less,_extension.less,_theme.less,_widgets.less,_module.less,)
* Demostrar la capacidad de anular o ampliar el Magento LESS
* Explicar los conceptos principales de LESS
* Demostrar la capacidad de aplicar estilo a los correos electrónicos
* Explicar la jerarquía de estilos ( lib, módulos, temas)
* Demostrar la capacidad de implementar y personalizar los componentes de la biblioteca LESS
* Identificar las diferencias entre la compilación del lado del cliente y la del lado del servidor y cómo funciona

Sección 4: JavaScript (utilidades de imagen, biblioteca de imágenes, módulo de datos del cliente, plantillas de Knockout) (22%)

* Demostrar la capacidad de inicializar y llamar a componentes de JavaScript
* Distinguir casos de uso para diferentes componentes de JavaScript
* Demostrar el uso de RequireJS
* Demostrar la capacidad de implementar diferentes tipos de mezclas
* Describir cómo añadir una traducción en JS
* Describir interacciones entre componentes de interfaz de usuario
* Demostrar el uso de Knockout JS
* Demostrar el uso de widgets jQuery
* Demostrar el uso de los componentes de JS mediante el uso de XML de diseño

Sección 5: Configuración de administración y PageBuilder (12%)

* Demostrar el uso de la administración de contenido
* Aplicar/programar un tema a un ámbito específico (sitio web, tienda, vista de tienda)
* Aplicar cambios de diseño a categorías, productos y páginas de CMS mediante la configuración de administración
* Describir la optimización del front-end
* Personalización de correos electrónicos transaccionales
* Demostrar el uso de las herramientas de desarrollo de administración

Sección 6: Herramientas (CLI y Grunt) (8%)

* Demostrar el uso de comandos básicos bin/Magento
* Describir el uso de los comandos del Compositor (instalar, actualizar, requerir, quitar)
* Diferenciación del caso de uso adecuado para los modos de implementación
* Definir la configuración y el uso de Grunt

Nuestras pruebas de práctica se desarrollan desde el mismo modelo que los exámenes en directo. La prueba de práctica le ayudará a comprender los temas tratados en el examen en vivo.

Recuerde que aprobar una prueba de práctica no garantiza que se aprobará el examen de certificación.

Vaya a la [Portal de preparación de examen de certificación de Adobe](https://www.certmetrics.com/adobe/candidate/gmetrix_sso.aspx){target="_blank"} ahora, para realizar una prueba de práctica.

**Haga el examen de certificación**

Adobe utiliza dos proveedores para administrar los exámenes procesados, PSI y Ejemplos.

* **PSI** ofrece opciones de centro de pruebas y exámenes japoneses (si está disponible)
* **[!DNL Examity]** ofrece una opción en línea remota

Para programar, volver a programar o cancelar el examen, simplemente haga clic en el proveedor que desee y siga las instrucciones que aparecen en la pantalla. ¡Es así de simple!
