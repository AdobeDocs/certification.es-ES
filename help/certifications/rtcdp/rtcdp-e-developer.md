### Detalles del examen {#exam-details}

* Nivel: Experto (1-3 años de experiencia)
* Puntuación superada: 45/68
* Tiempo: 136 min
* Entrega: En línea con protección (requiere acceso a la cámara)
* Idioma: inglés
* Costo: 225 dólares (global) / 150 dólares (India)
* Identificador del examen: AD0-E605

<br>

### Destinatarios previstos {#intended-audience}

* Desarrollador de software
* Ingeniero de datos
* Arquitecto de datos
* Arquitecto de soluciones
* Arquitecto empresarial

<br>

### Experiencia mínima {#minimum-experience}

Debería tener:

* Comprensión de las diferencias de evaluación de segmentos (flujo, lote, perímetro)
* Comprensión de los casos de uso de las API (cómo crear un proyecto de desarrollador, cómo funciona la autenticación)
* Comprensión del servicio de identidad
* Comprensión del flujo de datos
* Comprensión del Adobe I/O
* Comprensión del funcionamiento de las relaciones
* Capacidad para crear un modelo de datos (esquema, identidad, relaciones) para el caso de uso del cliente
* Capacidad para crear estrategias para combinar los datos en función de la política de combinación
* Comprensión de cómo determinar la forma correcta de etiquetado de datos, aplicación de políticas y Privacy Service
* Familiaridad con los límites del patrón de ingesta de datos
* Explicación de los destinos y qué utilizar
* Comprensión del marco de recopilación de datos Edge
* Capacidad para determinar qué permisos de AEP se deben configurar (es decir, permisos de nivel de atributo) (hacer recomendaciones a clientes)
* Comprensión básica de los casos de uso del servicio de consultas
* Comprensión básica del SDK de fuentes/destinos

Debe estar familiarizado con estos conceptos:

* Diagramas relacionales de entidad
* Administración/permisos
* API de Experience Platform
* Adobe Experience Platform Debugger
* SDK web y móvil

Debe sentirse cómodo en estos entornos:

* AEP
* Red perimetral

<br>

### Objetivos y ámbito del examen {#exam-objectives}

{{exam-objectives}}

**Arquitectura de datos (19 %)**

* En función de un escenario, traducir RDBMS a Adobe [!DNL RT-CDP]Modelo de datos NoSQL de
* Mostrar las prácticas recomendadas para modelar el perfil del cliente en tiempo real
* Diseñar la estrategia de identidad y las relaciones para el modelado

**Perfil del cliente en tiempo real (15 %)**

* Describir cómo funciona el ensamblado de perfiles
* Describir el funcionamiento del gráfico de identidad
* Describir la composición del perfil
* Diferenciación entre el perfil de Edge y el de Hub

**Ingesta de datos (13 %)**

* Describa las capacidades de ingesta de datos con CDP.
* Describir las capacidades de la ingesta de Edge
* Describir las técnicas avanzadas relacionadas con la arquitectura de ingesta de datos

**Segmentación (18 %)**

* Describa las diferentes formas de crear audiencias y segmentos dentro de CDP
* Explicar los distintos tipos de segmentación y cómo funcionan
* Muestre una comprensión de cómo aplicar casos de uso a la activación de segmentos/audiencias

**Activación (12 %)**

* Explicar diferentes patrones de activación dentro de CDP
* Explicar las barreras alrededor de la activación
* Describir la personalización en el sitio

**Gobernanza (9 %)**

* Aplicar prácticas recomendadas para la gobernanza en el Adobe [!DNL RT-CDP] implementación
* Describir cómo administrar los datos en los distintos servicios

**Administración (15 %)**

* Comprender las capacidades de monitorización
* Describir alertas
* Describir el uso diferente de las licencias
* Describir casos de uso para la habilitación de ABAC dentro del Adobe [!DNL RT-CDP]
