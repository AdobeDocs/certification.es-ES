# Recorrido de certificación para [!DNL Adobe Analytics] Profesional del desarrollador

Aprenda a convertirse en un certificado [!DNL Analytics] Profesional del desarrollador.

## Experiencia mínima

Debe tener:

* 6 a 12 meses de experiencia en [!DNL Adobe Analytics] implementación (web y aplicación móvil)
* Experiencia actuando como un jugador que admite equipos de implementación de Analytics en la comunicación entre TI y la empresa para comprender cómo se traducen los requisitos comerciales en una solución técnica (SDR) y según los requisitos proporcionados por el analista
* Comprender qué tipo de variable es necesaria, dónde y cómo recopilar los datos, diseñar el modelo de datos (capa de datos), configurar y mantener la solución de administración de etiquetas

Experimente con las siguientes tareas:

* Interpretación de un documento de diseño de soluciones (SDR) y comprensión del propósito de todas las [!DNL Adobe Analytics] variables recopiladas
* Explicación y trabajo con Adobe (Experience Platform) Launch
* Explicación de la configuración de la línea base [!DNL Adobe Analytics] dimensiones y eventos Nombre de página, Nombre del vínculo, Activity Map, etc., pero trabajará con desarrolladores expertos para más implementaciones de clientes: La asignación de variables sigue siendo propiedad de desarrolladores de nivel de experto
* Creación y configuración de reglas en Launch siguiendo el diseño proporcionado por un experto en AA
* Prueba y validación de los datos que se capturan en variables y eventos activados mediante depuradores de consola web o sistemas de aplicaciones móviles como charles logs
* Comprender qué es una capa de datos y saber cómo leer un objeto JavaScript
* Uso compartido de informes, segmentos y el proceso para solicitar acceso, pero no cómo administrar grupos de usuarios, protocolos de gobernanza y también administrar el SDR.
* Proporcionar orientación y configurar la solución móvil/SDK
* Explicación del marco general detrás del seguimiento de vídeo (basado en eventos) de la documentación de la base de conocimientos proporcionada
* Realización de informes básicos desde Analysis Workspace y comprensión de los resultados de datos disponibles en el [!DNL Adobe Analytics] Data Warehouse
* Experiencia básica con Adobe Experience Platform Launch (Adobe Launch)

>[!NOTE]
>
>Su certificación es válida durante dos años a partir del momento de su aprobación. Se recomienda utilizar la versión más reciente cuando sea posible, ya que cubre las capacidades de los productos más recientes.


**Objetivos y ámbito del examen**

Sección 1: Analytics en el ecosistema de Adobe Experience Cloud (14%)

* Identificar las funciones y capacidades de Adobe Experience Cloud ID
* Identificación de los usos para el lanzamiento de Adobe
* Identificar cómo habilitar/admitir [!DNL Adobe Analytics] etiquetas en el ecosistema de Adobe Experience Cloud

Sección 2: Estrategia y diseño de Analytics basados en una referencia de diseño de solución (12%)

* Dada una referencia de diseño de solución, identifique qué objetos de datos deben rellenarse
* Dado que forma parte de un documento de especificaciones técnicas, identifique cómo implementar la captura de datos

Sección 3: Implementación y configuración de Analytics (30%)

* Identificar los pasos a implementar [!DNL Adobe Analytics] Código
* Identificar funciones de código de implementación mínimo para el seguimiento de Analytics
* Identificar los pasos para configurar varios componentes dentro de [!DNL Adobe Analytics] IU
* Identificar las funciones de [!DNL Adobe Analytics] configuración de variables y grupos de informes
* Identificar las características de una capa de datos y los valores de la capa de datos
* Identificar las funciones de App Measurement para las funciones de seguimiento, como el seguimiento de vínculos y el seguimiento del comercio electrónico

Sección 4: Sistemas de administración de etiquetas (18%)

* Identificar si se han cumplido los requisitos mediante auditorías de etiquetas
* Identificar los pasos para configurar el etiquetado de sitios web con Adobe Launch
* En un escenario, identifique cómo habilitar, modificar y solucionar problemas de las extensiones en Adobe Launch
* Identificar cómo crear reglas y elementos de datos

Sección 5: Mobile Services y API (8 %)

* Identificar las funciones de [!DNL Adobe Analytics] API (fuente de datos, almacén, fuentes de datos, informes)
* Identificar las características de las reglas de procesamiento y los servicios móviles

Sección 6: Pruebas, validación y resolución de problemas (18%)

* Identificar el significado de errores comunes de JavaScript
* Identificar el significado de común [!DNL Adobe Analytics] errores de variables
* En un escenario, identifique las prácticas recomendadas de validación de prueba y depuración que deben seguirse
* Identifique cómo rastrear un [!DNL Adobe Analytics] llamada al servidor durante todo su ciclo vital (es decir, desde la solicitud de señalización hasta que se muestre en el informe)
